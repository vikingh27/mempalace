## Decision: Honest novelty assessment — lifecycle is the gap, not the system

- **Individual components are NOT novel**: file storage, directory hierarchy, 30/60/90 tiering, knowledge graphs, auto-save hooks — all standard or already exist in MemPalace
- **Claude Code already has**: Auto Memory, CLAUDE.md, Hooks (26 event types), Skills, /memory command
- **Why Claude Code team hasn't built it**: Auto Memory solves 80%; remaining 20% is deeply personal; MCP is their answer for power users; a full memory system is a product (15+ files, 6000+ lines), not a feature
- **The one genuinely unaddressed gap**: "Everyone building AI memory focuses on storage and retrieval. Nobody focuses on lifecycle and decay."
- **What we're building**: A lifecycle discipline layer on top of existing capabilities — not novel, but applied to a real, unaddressed gap
